POST
/api/v1/auth/refresh
Rotate refresh token

Validates the current refresh token against the sessions collection and performs token rotation: the old hashed RT is overwritten with a new one. This prevents replay attacks – each refresh token is single-use.

Parameters

No parameters

POST
/api/v1/auth/password/forgot
Step 1 — Forgot password: send OTP to registered email

Generates a 6-digit OTP valid for 10 minutes and dispatches it to the registered email address. The response is always identical regardless of whether the email exists (prevents user enumeration).

POST
/api/v1/auth/password/verify-otp
Step 2 — Verify OTP: exchange OTP for a reset token

Validates the 6-digit OTP from Step 1. On success, returns a reset token valid for 60 minutes that must be supplied in Step 3. The OTP is consumed immediately and cannot be reused.

/api/v1/auth/password/reset
Step 3 — Reset password: set new password, sign out all devices

Validates the reset token from Step 2, updates the password hash, and revokes every active session across all devices (Step 4 — re-sync). The user must sign in again with the new password.
